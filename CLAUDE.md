# מערכת ניהול מעבדת תיקונים — הורוביץ

## סקירה
מערכת ניהול מקצה לקצה למעבדת תיקונים של ציוד מטבח מוסדי וקירור מסחרי.
שני סוגי משתמשים: משרד (admin) ומעבדה (technician).
פיתוח שלבי לפי פרומטים מקצועיים. אסור לסטות מהאפיון.

## עקרונות מחייבים (אסור לסטות!)

### שפה ועיצוב
- **שפה:** עברית בלבד בכל ה-UI. הערות בקוד גם בעברית.
- **כיוון:** RTL מלא בכל מקום.
- **גופן:** Heebo בלבד (טעון מ-Google Fonts ב-index.css).
- **אייקונים:** lucide-react בלבד (יותקן בעת הצורך).

### תמחור ופיננסים
- **ללא מע"מ:** כל המחירים, החישובים והדוחות במערכת הם ללא מע"מ.
- **מע"מ רק בתצוגה:** מופיע *רק* במסך גביה כתצוגה ויזואלית (1000₪ + מע"מ 170₪ = 1170₪ כולל מע"מ).
- **מע"מ לא נכנס לחישובי רווח/הפסד.**

### שמירת נתונים
- **localStorage** עם prefix `horovitz_` לכל מפתח.
- **שמירה אוטומטית** על כל שינוי (debounced 500ms).
- אין שרת, אין DB — הכל בדפדפן.

### state management
- **Context API + useReducer** בלבד (לא Redux, לא Zustand).
- כל ישות עם reducer משלה.
- ה-store יושב ב-src/store/.

### תאריכים וכסף
- **תאריכים:** פורמט `DD/MM/YYYY HH:MM` (locale: he-IL).
- **כסף:** הצגה כ-"1500₪" (₪ אחרי המספר, ללא רווח).

## ארכיטקטורת ID

| ישות | פורמט | דוגמה |
|------|--------|--------|
| לקוח | `CUST-XXXX` | CUST-0042 |
| מכשיר (קבוע!) | `DEV-XXXX` | DEV-0017 |
| תיקון (לכל ביקור) | `QR_YYYYMMDD_XXX` | QR_20260615_007 |
| הזמנת רכש | `PO-XXX` | PO-0042 |
| ברקוד פנימי חלק | `BR-CAT-XXX` | BR-SENS-001 |
| קוד עבודה | `W-XXX` | W-001 |
| אצווה חלקים | `BATCH-XXX` | BATCH-0123 |

**חשוב:** קוד מכשיר קבוע לכל החיים של המכשיר, גם אם חוזר 10 פעמים לתיקון. בכל תיקון נוצר QR חדש.

## קונבנציות קוד

- **קומפוננטות:** PascalCase (UserCard.jsx)
- **hooks:** camelCase מתחיל ב-use (useRepairs.js)
- **utilities:** camelCase (calculateProfit.js)
- **constants:** UPPER_SNAKE_CASE
- **ID של ישות תמיד עם prefix** (ראה טבלה למעלה)
- **לא להשתמש ב-default export** מלבד דפים ראשיים ו-App.jsx
- **תיקיית src/components** היא לקומפוננטות נטו לשימוש חוזר. דפים ב-src/pages.
